- Reject `--all-packages` without `--refactor` because that PR would be too large.

- Keep startup behavior command-specific: `pr refactor` does not require LLM configuration or startup tool validation.
- Update tests around argument validation, generated PR-file instructions, feature names, and startup-validation bypass.
