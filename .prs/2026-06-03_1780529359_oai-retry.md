# PR

## User Summary (do not modify)

In this PR, add retry handling for OpenAI websocket disconnects.

Today, when the OpenAI connection drops, the TUI can surface a traditional Go chained error containing text like:

`stream ID x; INTERNAL_ERROR`

I think this likely comes from Go's HTTP streaming machinery somewhere underneath the OpenAI websocket/streaming path. The user-facing behavior should be better: if OpenAI disconnects the websocket connection mid-stream in a retryable way, Codalotl should retry the request somewhere appropriate instead of immediately surfacing that low-level transport error in the TUI.

Scope:
- Inspect the OpenAI streaming path, especially `internal/llmstream`, and find the right retry boundary.
- Add retry behavior for retryable OpenAI websocket/stream disconnects, including the `INTERNAL_ERROR`/stream-id failure mode if that is the actual error shape.
- Avoid retrying errors that are clearly non-retryable or would duplicate unsafe side effects.
- Make sure the TUI receives a sane final failure if retries are exhausted.

Validation:
- Add focused tests with testify for the retry classification and retry behavior.
- If request-shape or event replay tests change deliberately, manually patch those fixtures/tests to match.
- Run the relevant `internal/llmstream` tests.
- Run the relevant internal/noninteractive/integration tests against the mock OpenAI replay harness if this changes streamed OpenAI request/event behavior.

## Plan

### Package `internal/llmstream`
- Add OpenAI Responses streaming retry classification for retryable transport/stream disconnects, including stream `INTERNAL_ERROR` text from Go HTTP streaming errors.
- Ensure retry behavior happens at the existing conversation retry boundary and leaves the final TUI-facing failure sane if attempts are exhausted.
- Avoid retrying provider semantic errors such as `response.failed`, `response.incomplete`, and OpenAI stream `error` events unless they are explicitly transport-like.
- Add focused testify tests for retry classification and retry behavior with an OpenAI mock stream server.
- Run `go test ./internal/llmstream`.
