## Plan

### Phase 0: CLI argument design artifact [DONE]

No `SPEC.md` or implementation changes in this phase. Proposed user-facing command:

```text
codalotl cas ls-packages <namespace> [--csv] [--state=<state>] [--min-age=<duration>] [--min-churn=<percent>]
```

This replaces both existing commands:
- Delete `codalotl cas ls-stale`.
- Rename `codalotl cas ls-summary` to `codalotl cas ls-packages`.

#### Output columns

Keep the same five-column shape, but rename the confusing status columns:

```text
Package  Up to date  Stale  Age  Churn %
```

- `Up to date`: `yes` when the current package contents have a current record for `<namespace>`, otherwise `no`.
- `Stale`: `yes` when the package is not up to date but has a prior valid result for older contents, `no` when it has never had a valid result, and `-` when `Up to date` is `yes`.
- `Age`: for up-to-date packages, age of the current result; for stale packages, age of the prior valid result; `-` when there is no result.
- `Churn %`: approximate churn from the prior valid result to current contents; only meaningful for stale packages, otherwise `-`.

CSV output uses the same renamed headers.

#### `--state`

`--state` filters by the package's result state. Default is `all`.

Allowed values:
- `all`: every package discovered under the nearest git repo.
- `current`: only packages where `Up to date=yes`.
- `outdated`: packages where `Up to date=no`; this includes both `stale` and `missing`.
- `stale`: packages where `Up to date=no` and `Stale=yes`.
- `missing`: packages where `Up to date=no` and `Stale=no` because they have never had a valid result.

I prefer one `--state` flag over multiple boolean flags because it makes incompatible requests impossible and keeps help text short. If users need multiple groups, `outdated` covers the important combined case.

#### `--min-age`

`--min-age=<duration>` keeps rows whose displayed `Age` is at least the duration.

Duration syntax should be human-sized and compact, matching the table's intent:
- examples: `12h`, `30d`, `4w`, `1y`
- days are the primary expected unit

Rows with no age do not match `--min-age`.

#### `--min-churn`

`--min-churn=<percent>` keeps rows whose `Churn %` is at least the given percent.

Accepted forms:
- `--min-churn=20`
- `--min-churn=20%`

Rows with no churn do not match `--min-churn`.

#### Filter combination

Filters combine to optimize for the user's likely workflow: finding packages worth refreshing without rerunning work too often.

- `--state` selects the package population first.
- `--min-age` and `--min-churn` are threshold filters over that population.
- If both thresholds are supplied, both must match. This supports "only revisit hot packages after they are both old enough and changed enough".
- If a threshold flag is supplied without `--state`, use an implicit `--state=stale`, because churn is only meaningful against a stale prior result and this keeps `codalotl cas ls-packages specconforms --min-age=30d` focused.
- If `--state=outdated` is explicitly supplied with thresholds, missing packages remain included even though they lack age/churn. This keeps the "things needing first-time or refreshed work" workflow from hiding packages that have never been processed. Use `--state=stale` to apply thresholds only to packages with a prior result.

Examples:

```text
codalotl cas ls-packages specconforms
codalotl cas ls-packages specconforms --state=current
codalotl cas ls-packages specconforms --state=outdated
codalotl cas ls-packages specconforms --state=missing
codalotl cas ls-packages specconforms --min-age=30d
codalotl cas ls-packages specconforms --state=stale --min-age=30d --min-churn=20
codalotl cas ls-packages specconforms --state=outdated --min-age=30d --min-churn=20
```
