Background:
we have `codalotl docs add path/to/package` (docs everything) and `codalotl docs add --public-api path/to/package` (just public api) -- both for non-test only. However, I suspect we might want a mix of these.

Solution:
add `codalotl docs add --important path/to/package` -- document public api, as well as other "important" identifiers.

Design --important to be somewhat adjustable (with code changes), since i'm not 100% sure right now what important is. I may need to adjust it. Don't make decisions that lock in the definition where if I change it, we have to redo everything.

- everything in `--public-api`
- continue to exclude test unless `--include-test`
- all types. types includes their fields/methods
- every "big method/function":
    - threshold: 20 lines
- things with "high fan in"
    - threshold: ???
- things with "high fan out"
    - threshold: ???
- things in "big groups"
    - threshold: ???

both fan-in and fan-out are based on cyclic-groups. so a total of 5 incoming edges can be distributed to various identifiers in a group. but each identifier's fan in count is 5.

## Plan

### Phase 0: Important docs selection

#### Package internal/docubot [DONE]
- Add important-only documentation mode to `AddDocsOptions`.
- Centralize important identifier selection behind a small internal policy/helper so thresholds and category rules are easy to adjust with code changes.
- Important identifiers:
    - public/exported identifiers and package docs, matching public-only behavior
    - all types, including their fields and methods
    - functions/methods with at least 20 source lines
    - cyclic identifier groups with fan-in >= 3
    - cyclic identifier groups with fan-out >= 4
    - cyclic identifier groups with at least 3 identifiers
- Fan-in/fan-out/group-size use `gocodecontext.IdentifierGroup` after cyclic grouping, so group edge counts apply to every identifier selected from the group.
- Preserve normal test exclusion unless `DocumentTestFiles` is true.
- Add focused tests for selection behavior and option interactions.

#### Package internal/gocodecontext [DONE]
- Clarified `IdentifierGroup` API guidance for identifier-count vs context-size criteria while implementing docubot important selection.

#### Package internal/cli [DONE]
- Add `--important` to `codalotl docs add`.
- Treat `--important` and `--public-only` as mutually exclusive selection modes.
- Pass important mode to `docubot.AddDocs` and update help/tests.

#### Package internal/q/cli [DONE]
- Clarified `ArgsFunc` timing so command-specific flag-combination validation can rely on parsed flag values.

## Decisions

- Treat user wording `--public-api` as the existing `--public-only` flag.
- Initial threshold plan: big function/method >= 20 source lines, fan-in >= 3, fan-out >= 4, group size >= 3. These are code-owned defaults, not a hard API contract.
- Validation kept the initial thresholds. `internal/q/cli` selected a broad but plausible important set for a compact CLI framework package; `internal/gopackagediff` selected a small public/type/helper set.
