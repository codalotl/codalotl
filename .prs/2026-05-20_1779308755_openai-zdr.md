# PR

## User Summary (do not modify)

In this PR, we enable llmstream to support ZDR (zero data retention) for openai:
- First, kill NoLink. Instead, we rely on NoStore to mean:
    - if the provider doesn't store anything serverside anyway, is this field has no effect.
    - Otherwise, if it does store things server side by default (including linking when appropriate), setting NoStore will enable zdr mode, where we no longer store/link on the server.
- Second, make NoStore work, only for openai responses. If set, we set store=false, and don't link obviously. Each HTTP requset we make will have to include the whole history so that caching actually works.
- Don't set NoStore=true in production code by default. E.g., if I run `go run .` to enter TUI mdoe and use openai, it still uses NoStore=false (just like today).
    - But, you can and should test that NoStore=true actually works against openAI live servers. See below.
    - If $CODALOTL_ZDR=true then `go run .` (and so on) should use NoStore=true.

Implementing this is (probably) the easy-ish part. The hard part is validating it works. You (AIs) have a tendancy to just write code, see if tests pass, and call it a day. That won't work here. You must actually test this, with more than a single trivial request, that the real OpenAI actually works. Moreover, validate that input caching is working properly:
- The env has the real openai keys. Use them. Don't worry about sending live traffic to openai.
- make integration tests (see INTEGRATION_TEST) that exercise this.
- Test this with something like `go run . exec` with $CODALOTL_ZDR=true

## Plan

### Phase 0: design/spec [DONE]

Clarify the NoStore/ZDR semantics in the packages that own provider calls and agent session wiring.

#### Package `internal/llmstream` [DONE]
- Remove `SendOptions.NoLink`.
- For OpenAI Responses, `SendOptions.NoStore` means:
  - request includes `store=false`;
  - no `previous_response_id` is sent;
  - latest response ID is not retained for future linking;
  - every request includes full local conversation history so multi-turn/tool flows continue to work and prompt caching has the full prefix.
- Preserve default behavior (`NoStore=false`): store/link via OpenAI response IDs as today.
- Add focused request-shape tests and live `INTEGRATION_TEST=1` coverage that exercises multi-turn/tool ZDR and reports/validates input caching.

#### Package `internal/agent` [DONE]
- Add a construction option for no-store sessions.
- When enabled, every provider send from that agent uses `llmstream.SendOptions{NoStore: true}`.
- Subagents inherit no-store behavior from their parent agent.

#### Package `internal/agentregistry` / prepared agent boundary [DONE]
- Prefer carrying no-store policy via `agent.AgentCreator`, because `PreparedAgent.Create` delegates construction to the supplied creator and only passes registry-resolved options such as model.
- Add agentregistry/toolsetinterface API only if implementation proves the creator path cannot cover the root CLI/TUI entrypoints cleanly.

#### Packages `internal/noninteractive` and `internal/tui` [DONE]
- Default production behavior remains unchanged.
- `CODALOTL_ZDR=true` opts root CLI/TUI sessions into no-store agent construction.
- Noninteractive/tests can verify the env wiring and `go run . exec` can be used for live manual validation.

### Phase 1: implementation

Implement Phase 0 via subagents, likely in this order:
1. [DONE] `internal/llmstream`: remove NoLink, implement OpenAI NoStore request/history semantics, update unit/live integration tests.
2. [DONE] `internal/agent`: add no-store construction option and inheritance.
3. [DONE] Root creation plumbing: prefer passing a no-store-capable `agent.AgentCreator` into prepared agent creation; add agentregistry/toolsetinterface API only if needed.
4. [DONE] `internal/noninteractive` and `internal/tui`: read `CODALOTL_ZDR=true` for root sessions; add tests.

### Phase 2: validation

- Run targeted package tests after each implementation step.
- Run `INTEGRATION_TEST=1 go test ./internal/llmstream -run 'NoStore|ZDR|OpenAI'` with real OpenAI credentials.
- Run a live command similar to:
  `CODALOTL_ZDR=true go run . exec --yes --model <openai-model> 'make a small multi-step request that uses tools'`
- Inspect diagnostic/live output enough to confirm `store=false`, no linking, multi-turn success, and meaningful cached input on later requests.
