The AI agent added this line when fixing docs:

```json
docs_fix_result={"fix_count":1,"cas_namespace":"docs-fix-1","cas_record_path":"/Users/jonathannovak/leaf/code/multi_codalotl/codalotl_1/.codalotl/
      cas/docs-fix-1/63/cd17a6b70768f555828eb8fa257875cfd817435f19216b3647d7b49f036359","mode":"whole-package"}
```

This is insane. It shows up if the user types `codalotl docs fix internal/q/cli`.

The reason the AI wanted to do this (I think):
- it wanted the `refactor` tool to return JSON to LLM indicating any CAS files written.
- For other refactor subcommands, like dry, the refactor tool itself owns writing this.
- But for docs-fix, the tool itself owns writing it, because I wanted `codalotl docs fix internal/q/cli` to write CAS files, and the refactor tool just call this command.
- So it wrote this info to stdout, and then the refactor tool parses it. But the user sees it, and is confused about why internal messaging is printed for them.

First of all, it turns out that informing the LLM of CAS writes isn't actually important at all! So let's do the following:
- Edit the docubot package to not do this.
- make refactor not inform the LLM. The file will show up in git status, or it won't.
- Don't make refactor lie. Don't write `cas: null` or whatever, instead just skip that key.
