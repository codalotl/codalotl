## Plan

### Phase 0 [DONE] - Product CAS shape

- Updated `product-spec/features/cas.md` to make CLI namespace args non-versioned, keep filesystem namespace dirs versioned, and add `cas ls-namespaces`.

### Phase 1 - NamespaceSpec foundation

#### Package `internal/gocas`

- Add `NamespaceSpec` and `HashMode`.
- Convert `NamespaceSpec{Name, Version}` to existing versioned filesystem namespace strings so paths stay stable.
- Replace public package/code-unit methods that accept raw namespace strings with package-oriented methods accepting `NamespaceSpec`.
- `HashModePackage` hashes package Go files plus package-local `SPEC.md`.
- `HashModeCodeUnit` hashes the package's default Go code unit.

#### CAS-owner packages

- Keep `internal/gocas/casconformance`; it remains the owner for spec-conformance metadata and avoids duplicating this domain across CLI/TUI/tool packages.
- Expose namespace specs from CAS owners:
    - `internal/gocas/casconformance`: `specconforms`, v1, code-unit hash.
    - `internal/gocas/casclarify`: `clarify-public-api`, v1, package hash.
    - `internal/tools/refactor`: `refactor-*`, v1, code-unit hash.
    - `internal/cli`: `docs-fix`, v1, package hash.
- Update owners and callsites to use `gocas.NamespaceSpec`.

#### Package `internal/cli`

- Add a CAS namespace registry from the owning packages.
- Update `codalotl cas get <namespace>` to accept registered non-versioned namespace names and use the namespace's hash mode.
- Update existing `cas set` and `cas ls-unset` to use the same registered namespace resolution.
- Add `codalotl cas ls-namespaces`, sorted by namespace name, outputting current name + version and not hash mode.

### Phase 2 - Validation

- Run focused package tests for changed packages and `go test ./...`.
- Manually exercise:
    - `go run . cas ls-namespaces`
    - `go run . cas get <namespace> <path/to/pkg>`
    - `go run . exec ... check_spec_conformance ...`
- Commit any deliberately generated CAS record.
