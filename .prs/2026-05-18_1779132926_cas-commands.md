## User Summary (do not modify)

We are going to lay the foundation for `product-spec/features/cas.md`.

- Each package that owns a CAS-record should expose a `gocas.NamespaceSpec`, something in the direction of:

```go
var CASDryNamespace = gocas.NamespaceSpec{
    Name:     "dry",
    Version:  1,
    HashMode: gocas.HashModeCodeUnit,
}
```

- I don't think we need `internal/gocas/casconformance`? If no, delete it (move functionality into owning package).
    - If i'm wrong about that, and it's better to have it, that's fine, keep it. You decide!

- Update gocas public api to accept a `gocas.NamespaceSpec`

- keep filesystem paths of CAS records the same.
- keep internal/q/cas the same. q/cas doesnt care about versions.
- update `codalotl cas get <namespace>` to work with the new system.
- implement `codalotl cas ls-namespaces`
