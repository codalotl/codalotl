## User Summary (do not modify)

We're going to change how clarify improves documentation. I dislike how files "randomly" appear to sprout new comments. Surprising to the user.

New solution:
- clarify does not autoimprove docs
- instead, it writes a CAS file attached to the target package. Inlcuded metadata:
    - question
    - answer
    - package that originated the question
    - anything else?

A future PR (not this one) will build a tool/solution to analyze these cas files to actually improve docs.
