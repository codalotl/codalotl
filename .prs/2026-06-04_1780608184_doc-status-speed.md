## User Summary (do not modify)

Improve the performance of `codalotl docs status`. Keep behavior the same. Only implement an optimization if it doesn't increase technical debt.

## Plan

### Package `internal/cli`

- Optimize `codalotl docs status` without changing its table output, status semantics, package discovery behavior, or read-only behavior.
- Keep docs-add status based on `docubot.CountMissingDocs` with `docs add --important` selection.
- Keep docs-fix status based on current whole-package `docs-fix` CAS records.
- Optimize reflow status by avoiding one temporary clone per package. Prefer one shared temporary clone per module, clone each package into that module clone, run a batched dry-run reflow check over clone paths, and map modified clone files back to package rows.
- Preserve error behavior: packages that fail to load or fail status checks should still render `error` in the relevant columns rather than aborting the whole command where current behavior continues.
- Update focused `internal/cli` tests to prove behavior stays read-only and that reflow checking is batched/shared rather than per-package-clone.

No `SPEC.md` edits planned because `internal/cli/SPEC.md` already captures the external behavior and this PR should only change implementation strategy.
