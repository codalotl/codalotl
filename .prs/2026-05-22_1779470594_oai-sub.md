## User Summary (do not modify)

In this PR, we let enable codalotl to use the openai chatgpt subscription, just like all the other OSS agents do:
- use the client_id of app_EMoamEEZ73f0CkXaXp7hrann

The below is handwavy and is intended to show architecture, not specific method names or parameters.

New package: `internal/subscriptions/openaisub`
- contains the code that knows how to deal w/ openai subscriptions
- calls methods on llmmodel to set set subscription info

LLMStream does NOT depend on `internal/subscriptions/openaisub`. Instead it calls llmmodel to get provider subscription info.
- note: for openai, subscription info needs zdr (nostore). It also requires setting systeminstructions or whatever on root request, vs using an input array item.

CLI:
- `codalotl auth openai login`
- `codalotl auth openai login --no-browser`
- `codalotl auth openai logout`
- `codalotl auth openai status`

CLI startup (for TUI, for instance) calls ~ `openaisub.Init()` (or something better named, which):
- sees if `~/.codalotl/openai_auth.json` exists (path may be injected? see how we deal w/ this global config path elsehwere)
- If so and is valid, calls `llmmodel.SetSubscription(the info)`
