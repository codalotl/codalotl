## Review

### Review feedback [DONE]

- [P2] `internal/agentbuilder/data/config.yml`: the YAML-backed `review` tool requires findings to include `code_location.absolute_file_path`, but the `pr-review` subagent currently only receives git log/stat/diff output and no sandbox-root context. That means real findings are likely to emit schema-invalid or guessed paths instead of reliable absolute paths. The existing manual smoke test only covered the no-findings path.

Verdict: patch is incorrect. The new in-process review flow works for no-findings output, but it does not yet provide enough context for schema-compliant findings when issues are present.

Actioned by passing `sandbox_dir` into the built-in `review` subagent context with explicit absolute-path guidance, plus regression coverage in `internal/agentbuilder`.

## Decisions

- The new YAML/runtime primitives should be generic building blocks for in-process LLM tools, not a review-specific builtin.
- One required primitive is being able to send prompt/message content from text or `.md` files through the tool runtime; `review` should be expressed in terms of that capability.

## Learnings

- 2026-04-07: The prior manual validation only proved the no-findings path. We also need an end-to-end run that produces findings, because that is where raw JSON formatting regressions surface in user-visible output.

The outer `review` tool completion formats correctly as `Reviewed ...`, but the nested `pr-review` subagent still emits its raw JSON as a separate assistant-text line immediately beforehand, so users still see JSON in the live flow.
