## User Summary (do not modify)

Support the latest model from OpenAI (gpt-5.5).

per 1M tokens, $5 input, $30 output, $0.50 cached input
1,050,000 context window
128,000 max output tokens

Technically, pricing is split based on short vs long context (272k tokens). Pricing for long is double. ONLY do the short vs long thing if we already support it, otherwise don't.

## Plan

### Phase 0

In this phase, land the core model metadata update and make the OpenAI default track the newest supported GPT-5 large model.

#### Package internal/llmmodel
- Update `internal/llmmodel/SPEC.md` so OpenAI reasoning variants apply to the provider's default large GPT-5 model rather than a hardcoded `gpt-5.4`.
- Keep pricing flat in `ModelInfo`; for GPT-5.5 use the short-context prompt pricing and do not add new API/schema fields for the long-context premium.
- Implementation should add `gpt-5.5` metadata, make `gpt-5.5-high` the OpenAI default, and update focused `llmmodel` tests accordingly.

### Phase 1

In this phase, update downstream examples/tests that hardcode the old default model string.

#### Package internal/noninteractive
- Update hardcoded `gpt-5.4-high` expectations and integration fixtures that represent the repo default model.
- Update `SPEC.md` examples if they should reflect the default model after Phase 0.

#### Package internal/cli, internal/tui
- Update examples/tests that explicitly mention the old default model string.
- Adjust `SPEC.md` examples only where they are intended to show the default/recommended OpenAI model.

#### Package README.md, manual/MANUAL.md
- Refresh recommended/default model examples from `gpt-5.4-high` to `gpt-5.5-high` if they are meant to describe the current default.

### Validation
- Run focused package tests as each package lands.
- After implementation is complete, run `review` and `check_spec_conformance --only_changed`.
