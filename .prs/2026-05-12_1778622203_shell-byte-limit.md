## User Summary (do not modify)

Let's modify the shell tool as follows:
- default truncate to 40,000 bytes.
- shell tool has parameter to override this value. use units are bytes, not tokens.
    - value has some sensible ceiling (pick a static number - not parameteritized).
    - value also has some sensible floor (we don't want to handle edge cases like 1 byte)
- truncation specifics:
    - keeps head and tail (~evenly split)
    - middle has some sort of indicator that there's truncation, for example, `[... 32245 bytes elided ...]` - you pick a good indicator.
    - don't split utf8 bytes

in addition to normal Go tests, validate behavior by running something like `go run . exec`. Inspect relevant files in .logs to ensure right data is sent and elisions are done correctly.
